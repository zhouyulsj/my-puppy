# 毛球健康管家 - 竞赛注意事项

在项目根目录写一个简短的规则文件：
技术栈：单HTML文件，内嵌CSS+JS，Tailwind CSS CDN，原生JS
禁止：React/Vue/框架、外部JS/CSS文件、npm/构建工具
风格：中文UI，暖色系(#FF6B35主色)，移动端优先
结构：两视图切换，4个Tab，AI API调用

## 1. API Key 提前准备好
竞赛现场再注册API会浪费时间。提前注册智谱/DeepSeek/通义任一平台，拿到Key，测试一次能调通。建议备两个平台的Key，防一个挂了。

## 2. 每步Prompt执行后立即验证
不要7步全跑完再测试。每步完成后：
- 浏览器打开看页面是否正常渲染
- F12控制台看有无JS报错
- 有错就当场让AI修，不要攒着

## 3. 最容易出错的3个环节

| 环节 | 常见问题 | 预防 |
|---|---|---|
| AI返回的JSON解析 | AI不总是严格按格式输出，多出markdown代码块标记 | 在callAI中加清洗逻辑：去掉```json和```包裹，trim后再parse |
| 品种联动 | 切换猫/狗后品种没更新，或更新了但选中项残留 | 切换species时先清空breed的value再填充options |
| localStorage | 存入的是对象但取出时忘了JSON.parse | 封装统一的getStorage/setStorage函数 |

## 4. 给每个Prompt加一句"保险栓"

在Prompt 3的AI调用函数中，JSON解析前加这段：

```javascript
let content = response.choices[0].message.content.trim();
content = content.replace(/^```json\s*/, '').replace(/\s*```$/, '');
```

## 5. 竞赛现场策略

- **先跑通骨架**：Prompt 1完成后确认页面能打开、表单能填、能切换视图，这是底线
- **Prompt 2-3是数据层**：不涉及UI，出错概率低
- **Prompt 4-6按优先级做**：健康评估 > 采购清单 > 全年日历。如果时间不够，砍日历
- **Prompt 7最后做**：打磨是锦上添花，功能完整比好看重要

## 6. 准备一个"现场急救Prompt"

如果某步AI输出有bug，直接把报错信息+相关代码片段丢给AI：

```
以下代码运行时报错：[粘贴报错信息]
相关代码：[粘贴代码片段]
请只修复这个错误，不要重写其他部分。
```

## 7. 单文件HTML的坑

- **Tailwind CDN在弱网下加载慢**：提前下载tailwind.min.js到本地备用
- **中文字体加载慢**：Noto Sans SC可以换成系统字体`-apple-system, "Microsoft YaHei", sans-serif`，不加载Google Fonts
- **文件太大浏览器卡**：如果最终HTML超过3000行，考虑把JS逻辑拆成`<script>`标签分块写，方便定位

## 8. 评审视角的加分项

- 页面底部加一行 **"由TRAE辅助生成"**，既诚实又展示AI协作能力
- 准备一个30秒的口头介绍话术：**"问题→方案→技术→亮点"**四句话讲清楚
